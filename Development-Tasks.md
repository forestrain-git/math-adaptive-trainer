# Math Adaptive Trainer — 开发任务清单

**版本：** MVP v0.1.0  
**日期：** 2026-04-29  
**对应 PRD：** Product-Spec.md v0.1.0

---

## 总览

本文档将 Math Adaptive Trainer MVP 的开发工作拆分为 22 个阶段，共约 55 个具体任务。每个任务包含编号、名称、描述、输入、输出、依赖、验收标准、优先级和复杂度评估。

**优先级定义：**
- **P0：** 阻塞后续任务，MVP 不可用即失败
- **P1：** MVP 核心体验，缺失会严重减分
- **P2：** 增强体验，可做可不做

**复杂度定义：**
- **低：** 1-4 小时可完成
- **中：** 半天-1 天可完成
- **高：** 1-2 天可完成

## 阶段 7：每 25 题动态调整引擎

### 任务 7.1：实现 analyzeCurrentWeakSkills

- **任务描述：** 分析已完成答题记录，判断哪些 skillTag 是当前弱项。
- **输入：** PRD 第 15.1 节弱项判断规则 + QuestionRecord[]
- **输出：** `src/engine/adaptiveEngine.ts` 中的 `analyzeCurrentWeakSkills` 函数
- **依赖任务：** 3.3, 5.1
- **验收标准：**
  - [ ] 正确率低于 70% 的 skillTag 被标记为弱项
  - [ ] 平均用时超过目标用时 1.5 倍的 skillTag 被标记为弱项
  - [ ] 同一 skillTag 连续错误 2 次以上被标记为弱项
  - [ ] 返回 SkillTag[]
- **优先级：** P0
- **预计复杂度：** 中

### 任务 7.2：实现 analyzeCurrentStrongSkills

- **任务描述：** 分析已完成答题记录，判断哪些 skillTag 是当前强项。
- **输入：** PRD 第 15.2 节强项判断规则 + QuestionRecord[]
- **输出：** `src/engine/adaptiveEngine.ts` 中的 `analyzeCurrentStrongSkills` 函数
- **依赖任务：** 3.3, 5.1
- **验收标准：**
  - [ ] 正确率不低于 85%、平均用时不超过目标用时 1.2 倍、样本不少于 3 题才标记为强项
  - [ ] 样本量少于 3 题不强项判断
  - [ ] 返回 SkillTag[]
- **优先级：** P1
- **预计复杂度：** 中

### 任务 7.3：实现 buildSkillDistributionForNextBlock

- **任务描述：** 根据弱项和强项分析结果，构建下一组 25 题的题型分布配置。
- **输入：** PRD 第 10.3-10.7 节动态调整规则 + weakTags + strongTags
- **输出：** `src/engine/adaptiveEngine.ts` 中的 `buildSkillDistributionForNextBlock` 函数
- **依赖任务：** 7.1, 7.2
- **验收标准：**
  - [ ] 返回 QuestionConfig[]，总题数恰好 25
  - [ ] 弱项最多占 60%（15 题）
  - [ ] 已掌握题型至少保留 20%（5 题）
  - [ ] 随机复习题占 10%-20%（3-5 题）

- **预计复杂度：** 高

## H. 主要开发风险和规避方式

| 风险 | 影响 | 规避方式 |
|------|------|----------|
| HashRouter 路径与预期不符 | 高 | 开发初期就配置好路由，每次开发都通过 `/#/path` 访问测试 |
| Vite base 配置错误导致资源 404 | 高 | 2.1 任务尽早完成，build 后立即检查 index.html 中的资源路径 |
| IndexedDB 在 MatePad 浏览器上不可用 | 高 | 4.2 中加入可用性检测，不可用时给出明确提示 |
| 出题算法死循环或性能差 | 中 | 6.2 中设置最大尝试次数，避免无限循环 |
| 动态调整算法分配题量不为 25 | 中 | 7.3 中加入断言检查，buildDistribution 后验证总数 |
| 组间休息页状态丢失 | 中 | 16.4 中确保 records 在组件生命周期中持续保存 |
| 训练数据未正确保存导致结果页空白 | 高 | 9.3 和 16.5 中加入保存异常处理，跳转前确认保存成功 |
| 掌握度更新计算错误 | 中 | 11.1 中编写单元测试验证公式正确性 |
| MatePad 虚拟键盘遮挡输入框 | 高 | 19.2 中测试虚拟键盘弹出时的布局，必要时滚动页面 |
| GitHub Pages 缓存导致更新不及时 | 中 | 在 URL 后加 `?nocache=1` 强制刷新，或等待缓存过期 |
| 项目构建后体积过大 | 低 | 关注 build 输出大小，必要时开启代码分割 |
| 跨浏览器 IndexedDB 兼容性问题 | 中 | 测试 Chrome 和 MatePad 自带浏览器，使用 Dexie.js 封装降低兼容风险 |
